MCP Security Considerations

  • Tool approval — Require explicit user consent before agent invokes tools
  • Least privilege — MCP servers should only expose necessary capabilities
  • Transport security — Use stdio for local, SSE/Streamable HTTP for remote with TLS
  • Input validation — MCP servers must validate all inputs from the model
  • Prompt injection — Untrusted data in tool results can manipulate agent behavior
Security is critical when giving AI agents access to tools and systems. MCP provides the mechanism, but you need to configure it securely.

Securing the Agent Toolchain

  • Allowlist MCP servers — Only approved servers in your configuration
  • Audit tool usage — Log all MCP tool invocations for review
  • Sandbox execution — Run MCP servers in containers or restricted environments
  • Policy as Code — Enforce guardrails via automated policy checks
  • Supply chain — Vet MCP server dependencies like any other software
Apply the same supply chain security practices to MCP servers that you would to any third-party dependency.
