  • Host вҖ” The application (IDE, agent) that connects to MCP servers
  • Client вҖ” Maintains 1:1 connection with each server
  • Server вҖ” Exposes tools, resources, and prompts
MCP uses a client-server architecture. The host (IDE/agent) connects to multiple MCP servers, each providing access to different tools and data sources.

MCP Capabilities

Capability Description Example
Tools Functions the model can invoke Run a security scan, create a PR
Resources Data the model can read Database schemas, API docs
Prompts Reusable prompt templates Code review checklist
MCP servers expose three types of capabilities: tools, resources, and prompts.
