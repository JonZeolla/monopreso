What is TLS

The primary goal of TLS is to provide a secure channel between two communicating peers; the only requirement from the underlying transport is a reliable, in-order data stream.
- David Wong

What is TLS

TLS allows client/server applications to communicate over the Internet in a way that is designed to prevent eavesdropping, tampering, and message forgery.
- RFC 8446

Typical Goals of using TLS

  • Nobody else can see this
  • We are who we say we are
  • The data didn't change
  • Refer to this great post on TLS and related attacks/downsides.

Typical server setup

  1. Create associated public (CSR, CRT, or Cert) and private (PEM or Key) information
  2. Get your public information endorsed by a trusted third party
  3. Publish that endorsement
  4. Configure the server to make communications with your clients private
Certificates
  • Certificates are X509v3 files, and one of their elements is the public key, which is signed by a CA. There are numerous other elements of a certificate to hold information like the Organization name, etc. There are also x509v3 extensions to hold things like the subjectAltName (SAN)
    • The public key is embedded in the X509 certificate RFC2459 in section 4.1.2.7
  • The private key is associated with the public key using assymetric crypto
  • CAs are preloaded in client OSs and/or browsers as a 'trusted third party'
    • X509v3 came out in 1999 https://tools.ietf.org/html/rfc2459
    • In TLS 1.3 using x509v3 certificates are still preferred, although it's possible to negotiate a raw key via RFC7250
  • Extensions have an OID and an ASN.1 structure https://tools.ietf.org/html/rfc2459#section-4.2
