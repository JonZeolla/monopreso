HTTP/2

  1. TLS is not required, but many browser vendors require it.
  2. When TLS is in use, HTTP/2 mandates TLS 1.2 or newer, SNI, as well as disabling renegotiation and compression.
    • It also strongly recommends a list of more secure cipher suites.
  • There are some additional security considerations, some of which apply to TLS and HTTP/2. For instance, HTTP/2 over TLS requires the use of ALPN which is a relatively new TLS extension, RFC 7301, which helps us negotiate the new HTTP version without losing valuable time or network packet round-trips. ALPN came from NPN (Next Protocol Negotiation). ALPN primarily differs from NPN in who decides what protocol to speak. With ALPN the client tells the server a list of protocols in its order of preference and the server picks the one it wants, while with NPN the client makes that final choice.
