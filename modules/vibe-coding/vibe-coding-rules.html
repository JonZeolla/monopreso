Creating an AI-First Codebase

  • Test input data is key; have diverse, complete examples of scenarios and use them in tests. AI tools will update tests to pass, masking bugs.
  • If you expect something to always be the case in your code, enforce it with code.
  • Check for the same thing multiple ways. AI can be tricky or work in unexpected ways.
  • Prefer semantically aware checks (like semgrep) over specific structure tests (like regex/grep) where you can.
  • Optimize your code to be easily deleted
